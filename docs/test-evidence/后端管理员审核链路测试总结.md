# 后端管理员审核链路测试总结

## 一、测试目的

本轮测试用于验证“清华食堂点评助手”后端中的管理员审核业务链路是否完整可用。前一轮 `smoke-test.ps1` 已经验证了公告、食堂列表、推荐菜品、注册登录、JWT 鉴权、拥挤度上报、收藏等基础接口。本轮进一步补充测试管理员相关接口，重点验证以下功能：

1. 普通用户可以正常注册、登录并获得 JWT Token。
2. 普通用户可以提交新菜品。
3. 用户提交的新菜品不会直接进入正式菜品列表，而是进入待审核状态。
4. 管理员用户可以通过 `/api/admin/submissions` 查看待审核菜品。
5. 管理员可以审核通过用户提交的菜品。
6. 审核通过后，菜品进入正式菜品列表，并可通过普通菜品搜索接口查询。
7. 普通用户可以发布评论，菜品详情接口能够正确返回评分和评论数统计结果。

## 六、测试结论

本轮管理员审核链路测试通过。测试结果表明，后端已经实现并跑通了从普通用户提交菜品到管理员审核，再到正式菜品展示的完整业务闭环。

具体来说，普通用户提交的新菜品首先进入 `PENDING` 状态，不会直接出现在正式菜品列表中；管理员用户在重新登录并获得 `ADMIN` 权限 Token 后，可以查看待审核提交并执行审核通过操作；审核通过后，后端会生成正式菜品记录，并且该菜品可以通过普通菜品搜索接口查询到。

此外，本轮测试还验证了评论发布和评分统计逻辑。普通用户发布评分评论后，菜品详情接口能够返回更新后的 `avgRating` 和 `reviewCount`，说明评论数据写入和统计聚合逻辑均能正常工作。

因此，后端在以下方面已经具备基础可交付状态：

```text
1. 用户注册与 JWT 登录鉴权
2. 普通用户提交菜品
3. 管理员权限识别
4. 管理员待审核列表查询
5. 管理员审核通过
6. 审核后菜品正式展示
7. 评论发布
8. 菜品评分与评论数统计
```

---

## 七、注意事项
1. 测试输出中包含 JWT Token。正式报告截图时建议对 Token 长串进行打码，避免展示完整 Token。
2. 部分接口在刚创建对象后返回的 `createdAt`、`updatedAt` 为 `null`，但后续重新查询时可以看到时间字段。这通常是因为接口返回的是插入时的内存对象，没有重新从数据库查询自动填充后的记录，不影响本轮功能测试结果。
3. 当前管理员账号通过 MySQL 手动修改 `users.role` 得到，适合课程项目测试和演示。后续如需完善系统，可以增加管理员初始化脚本或专门的后台管理账号配置。
4. 本轮测试主要验证“审核通过”链路，后续如需进一步增强测试完整性，可以补充“审核驳回”链路，即调用 `/api/admin/submissions/{id}/reject` 并确认驳回后的菜品不会进入正式菜品列表。

### 附录 D. 管理员审核链路测试命令

本部分用于测试“普通用户提交菜品 -> 管理员查看待审核列表 -> 管理员审核通过 -> 菜品进入正式列表”的完整链路。

#### 1. 设置后端地址
```powershell
$BASE = "http://localhost:8080"
```

---

#### 2. 注册并登录普通用户
```powershell
$normalUsername = "submitter_$(Get-Date -Format 'MMddHHmmss')"

$normalRegisterBody = @{
    username = $normalUsername
    password = "123456"
    nickname = "审核测试普通用户"
} | ConvertTo-Json

Invoke-RestMethod `
  -Method Post `
  -Uri "$BASE/api/auth/register" `
  -ContentType "application/json" `
  -Body $normalRegisterBody
```

登录普通用户：
```powershell
$normalLoginBody = @{
    username = $normalUsername
    password = "123456"
} | ConvertTo-Json

$normalLogin = Invoke-RestMethod `
  -Method Post `
  -Uri "$BASE/api/auth/login" `
  -ContentType "application/json" `
  -Body $normalLoginBody

$normalToken = $normalLogin.data.token
$normalHeader = @{ Authorization = "Bearer $normalToken" }

$normalLogin | ConvertTo-Json -Depth 10
```

---

#### 3. 普通用户提交新菜品
