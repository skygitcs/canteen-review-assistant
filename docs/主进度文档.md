# 进度报告

更新时间：2026-06-07

## 1. 项目阶段

本项目的初始想法在 2026-04-17 提出。当时小组注意到，清华校内缺少一个集中查看食堂、窗口、菜品、评价和公告信息的移动端应用，因此确定以“THU canteen / 清华食堂点评助手”作为课程大作业选题。

根据 Git 提交记录，仓库正式开发从 2026-05-13 开始，随后经历了项目结构搭建、Android 原型开发、后端接口设计、前后端联调、真实数据导入、图片上传、评论互动、管理员审核、地图和文档测试补充等阶段。当前项目已完成课程大作业的主要闭环：

- Android 客户端：登录注册、首页、全部食堂、地图、食堂详情、菜品详情、个人中心和管理员后台。
- 后端服务：鉴权、食堂、菜品、评论、收藏、补充菜品、审核、公告、上传和拥挤度接口。
- 数据库：核心业务表、种子数据和静态图片资源。
- 文档：启动说明、用户手册、接口文档、实现说明、测试报告和评分项对照。
- 测试：新增 Android JVM 单元测试，后端保留冒烟测试脚本。

## 5. 风险与处理

| 风险 | 当前处理 |
| --- | --- |
| 真机无法访问本机后端 | 使用 `adb reverse tcp:8080 tcp:8080`，文档中提供排查步骤 |
| 中文种子数据导入乱码 | MySQL 导入时使用 `--default-character-set=utf8mb4` |
| 新菜品未审核直接展示 | 设计为先进入 `dish_submission`，管理员通过后再进入正式菜品 |
| 评论或收藏重复提交 | 后端业务校验和统一异常响应处理 |
| 空数据导致页面不清晰 | Android 列表页展示空状态 |
| 自动化测试无法执行 | 当前缺少 `gradle-wrapper.jar`，测试报告已记录恢复方式 |
