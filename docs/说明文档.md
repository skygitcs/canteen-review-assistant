# 清华食堂点评助手实现说明文档

更新时间：2026-06-07

## 1. 项目概述

清华食堂点评助手面向校内食堂浏览、菜品查询、评价分享、收藏管理和菜品补充审核等场景。项目采用 Android 客户端 + Spring Boot 后端 + MySQL 数据库的结构，Android 端负责移动交互体验，后端负责数据存储、业务校验、鉴权和接口服务。

## 3. Android 客户端实现

### 3.1 登录注册

登录注册入口由 `AuthActivity` 承担，用户可使用演示账号登录，也可注册新账号。登录成功后后端返回 JWT，客户端保存在 `NetworkClient` 的 SharedPreferences 中，后续请求通过 OkHttp 拦截器自动加上 `Authorization: Bearer <token>`。

### 3.2 首页

首页由 `HomeFragment` 实现，主要包含：

- 顶部公告栏：使用 `ViewPager2` 纵向滚动展示公告，多个公告时自动轮播。
- 推荐食堂：按评分获取食堂列表，展示推荐食堂封面、名称、地址、评分和标签。
- 推荐菜品：通过推荐接口获取菜品列表，点击可进入菜品详情页。

### 3.3 全部食堂与地图

全部食堂由 `FoodMapFragment` 实现，提供列表视图和地图视图：

- 列表视图支持关键词搜索和标签筛选。
- 地图视图使用 `PhotoView` 展示校园食堂位置图，支持缩放和平移。
- 地图标记点随图片矩阵变化重新定位，点击标记后弹出食堂预览并可跳转详情页。
- 拥挤度热力数据通过后端接口获取并在列表区域展示。

### 3.4 食堂详情

`CanteenDetailActivity` 展示单个食堂的详情数据：

- 展示食堂封面、名称、地址和标签。
- 左侧楼层列表根据后端窗口数据动态生成。
- 窗口下拉筛选与楼层联动。
- 支持按菜名、窗口、描述和标签搜索。
- 支持按菜品标签筛选。
- 菜品列表点击后进入菜品详情页。
- 支持拥挤度上报。
- 支持用户补充新菜品信息。

### 3.5 菜品详情

`DishDetailActivity` 展示菜品详情：

- 菜品图片、名称、所属食堂、价格、描述和标签。
- 点击图片进入大图查看，支持缩放。
- 展示用户评论列表。
- 支持收藏和取消收藏。
- 支持发布新评论，提交后先本地即时追加，再刷新后端正式数据。
- 评论支持点赞、点踩和取消投票。

### 3.6 个人中心

`ProfileFragment` 负责个人信息与用户行为入口：

- 展示用户账号、昵称、角色和口味偏好。
- 支持编辑个人信息和口味偏好。
- 展示收藏菜品并可跳转详情。
- 展示本地活动记录。
- 管理登录状态。

### 3.7 管理后台

`AdminDashboardActivity` 提供管理员审核入口：

- 管理员可查看用户补充的菜品提交。
- 可查看提交详情，包括食堂、窗口、价格、辣度、描述、标签和图片。
- 可通过或驳回提交。
- 可发布公告，用于首页滚动展示。

## 4. 后端实现

后端按 Controller、Service、Mapper、Entity、DTO 分层：

| 模块 | 说明 |
| --- | --- |
| 鉴权模块 | 注册、登录、JWT 签发与解析、接口权限控制 |
| 食堂模块 | 食堂列表、详情、窗口、拥挤度上报、热力统计 |
| 菜品模块 | 菜品列表、详情、推荐、标签、搜索筛选 |
| 评论模块 | 发布评论、评论列表、点赞点踩 |
| 用户模块 | 个人资料、口味偏好、收藏增删查 |
| 提交审核模块 | 用户补充菜品信息、管理员审核、审核通过后写入正式菜品 |
| 公告模块 | 管理员发布公告、客户端首页读取公告 |
| 上传模块 | 图片上传和静态资源访问 |

后端通过 `GlobalExceptionHandler` 统一处理业务异常和参数校验异常，通过 `ApiResponse` 统一返回：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

## 5. 数据设计

核心数据包括：

| 数据 | 用途 |
| --- | --- |
| `canteen` | 食堂基础信息、位置、评分与拥挤度 |
| `canteen_window` | 食堂楼层与窗口 |
| `dish` | 菜品基础信息、价格、图片、辣度、统计字段 |
| `dish_tag` | 菜品标签，用于筛选和展示 |
| `review` | 用户评论、评分、图片 |
| `review_vote` | 评论点赞、点踩和取消投票 |
| `favorite` | 用户收藏菜品 |
| `dish_submission` | 用户补充菜品的待审核数据 |
| `announcement` | 首页公告 |
| `crowd_report` | 拥挤度上报数据 |

数据处理重点：

- 食堂、菜品、评论、用户、标签、收藏等数据可完整存储与读取。
- 菜品平均评分、评论数、收藏数、标签等统计支撑排序、筛选和推荐。
- 用户上传菜品进入审核流，避免未审核内容直接进入正式展示。
- 对登录态、非法输入、重复收藏、空数据和网络失败给出处理或提示。

## 6. 交互与体验

应用围绕移动端常见操作设计：

- 底部导航在首页、美食广场、个人中心之间切换。
- 食堂与菜品卡片支持点击跳转。
- 搜索框、标签 Chip、楼层列表、窗口 Spinner 共同完成筛选。
- 图片和地图支持缩放手势。
- 网络失败、提交失败、空数据等状态通过 Toast 或空状态文案提示。
- 首页公告自动滚动，满足课程评分表中的创意项。

## 7. 鲁棒性与限制

已处理：

- JWT 登录态自动附加和清理。
- 后端统一异常响应。
- 空数据列表展示空状态。
- 食堂、菜品、评论等接口失败时避免应用崩溃。
- 用户补充菜品需经过管理员审核。

当前限制：

- Android 客户端主要使用原生 Activity/Fragment，尚未引入 ViewModel 和 Repository 分层。
- Android UI 自动化测试尚未加入，当前新增的是可快速执行的 JVM 单元测试。
- 本仓库 Gradle wrapper 缺少 `gradle-wrapper.jar`，需要恢复后才能直接用 `gradlew.bat` 执行测试。
- 推荐逻辑当前以评分、统计和后端推荐接口为主，未实现复杂个性化算法。

```powershell

$env:DB_PASSWORD="你的MySQL密码"
```

```powershell
adb devices
adb reverse tcp:8080 tcp:8080
```
