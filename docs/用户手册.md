# 用户手册

本文档面向课程验收、报告撰写和组内测试，说明“清华食堂点评助手”的主要使用方式。

## 1. 应用简介

清华食堂点评助手用于帮助用户浏览校园食堂、查找菜品、查看评分评论、收藏菜品，并补充新的菜品信息。管理员可以审核用户提交的新菜品，让真实数据逐步进入系统。

## 2. 运行前准备

后端需要先启动：

```powershell
cd E:\canteen-review\canteen-review-assistant\backend
$env:DB_PASSWORD="你的MySQL密码"
mvn spring-boot:run
```

后端启动成功后，可以访问：

```text
http://localhost:8080/swagger-ui.html
```

Android 真机测试时，推荐使用 `adb reverse`：

```powershell
adb devices
adb reverse tcp:8080 tcp:8080
```

如果 PowerShell 找不到 `adb`，可以用完整路径：

```powershell
& "E:\AndroidSDK\platform-tools\adb.exe" devices
& "E:\AndroidSDK\platform-tools\adb.exe" reverse tcp:8080 tcp:8080
```

## 3. 登录和注册

打开应用后，用户先进入登录/注册界面。

可以使用演示账号：

| 用户名 | 密码 | 角色 |
| --- | --- | --- |
| demo | password | 普通用户 |
| reviewer | password | 普通用户 |
| admin | password | 管理员 |

也可以注册新账号。当前项目用于课程演示，注册信息保存到本地 MySQL 数据库。

## 4. 首页

首页展示应用的主要入口和公告信息。

用户可以在首页查看：

- 系统公告。
- 推荐菜品。
- 热门或常用功能入口。

公告栏会以滚动文字形式展示最近公告。

## 5. 美食广场

美食广场用于展示所有食堂。

当前包括：

- 紫荆
- 桃李
- 清芬
- 玉树
- 听涛
- 观畴
- 丁香

点击食堂卡片可以进入食堂详情页。即使某些食堂暂时没有菜品数据，也会保留食堂入口，页面中显示“暂无数据”。

## 6. 食堂详情

食堂详情页主要用于按楼层和窗口浏览菜品。

用户可以：

- 在左侧切换楼层。
- 选择窗口筛选菜品。
- 使用搜索框按菜名或关键词搜索。
- 使用标签筛选菜品。
- 点击菜品进入详情页。

如果某个楼层或窗口暂无菜品，会显示“暂无数据”提示。

## 7. 菜品详情

菜品详情页展示菜品的完整信息。

包括：

- 菜品图片。
- 所属食堂和窗口。
- 价格。
- 标签。
- 平均评分。
- 评论数量。
- 用户评论。

价格为 `0` 的菜品会显示为“称重计价”。

用户可以在详情页：

- 收藏或取消收藏。
- 写评价。
- 用五颗星给菜品打分。
- 查看其他用户评论。
- 对评论点赞或点踩。

## 8. 写评价

用户登录后可以给菜品写评价。

评价内容包括：

- 五星评分。
- 文字评价。
- 可选图片。

提交后，菜品的平均评分和评论数会更新。

## 9. 补充菜品信息

用户可以在食堂详情页补充新的菜品。

需要填写：

- 菜品名称。
- 所属食堂。
- 所属窗口。
- 价格。
- 简短描述。
- 辣度。
- 标签。
- 可选图片。

标签从系统已有标签中选择，点击后会以选中状态显示。图片会通过后端上传接口保存，上传成功后提交内容进入管理员审核列表。

普通用户提交的新菜品不会立即出现在正式菜品列表中，需要管理员审核通过。

## 10. 个人页

个人页展示用户相关信息。

用户可以：

- 查看昵称和账号。
- 编辑口味偏好。
- 查看收藏菜品。
- 点击收藏菜品进入详情。
- 管理登录状态。

口味偏好以独立标签块展示，多个偏好会自动换行。

## 11. 管理员功能

管理员账号登录后，可以进入管理后台。

管理员可以：

- 查看待审核的新菜品提交。
- 查看提交详情，包括菜品图片、食堂、楼层、窗口、价格、辣度、描述和标签。
- 批准提交。
- 驳回提交。
- 发布公告。

审核通过后，提交的菜品会进入正式菜品数据，并可在食堂详情和菜品列表中看到。

## 12. 常见问题

### 12.1 登录提示连接失败

先确认后端是否启动：

```text
http://localhost:8080/swagger-ui.html
```

真机测试时还要确认已经执行：

```powershell
adb reverse tcp:8080 tcp:8080
```

### 12.2 真机找不到后端

可能原因：

- 没有打开 USB 调试。
- 没有允许手机上的 USB 调试授权。
- 没有执行 `adb reverse`。
- 后端没有启动。
- App 中的 `BASE_URL` 与当前测试方式不匹配。

### 12.3 图片上传失败

可能原因：

- 后端没有重启到包含上传接口的版本。
- 图片超过上传大小限制。
- 手机没有授予图片选择权限。

当前后端单张图片上传上限为 10MB。

### 12.4 数据为空

通常是数据库没有导入种子数据。

进入 MySQL 后执行：

```sql
DROP DATABASE IF EXISTS thu_canteen;
SOURCE E:/canteen-review/canteen-review-assistant/backend/src/main/resources/db/schema.sql;
SOURCE E:/canteen-review/canteen-review-assistant/backend/src/main/resources/db/seed.sql;
```

导入中文数据时建议使用：

```powershell
mysql --default-character-set=utf8mb4 -u root -p
```

## 13. 验收演示建议流程

建议按下面顺序演示：

1. 启动后端，打开 Swagger 页面说明接口存在。
2. Android 端登录普通用户。
3. 浏览首页公告和推荐菜品。
4. 进入美食广场，查看所有食堂。
5. 进入某个食堂详情，按楼层、窗口和标签筛选菜品。
6. 打开菜品详情，查看评分和评论。
7. 写一条评价并打五星。
8. 收藏菜品，在个人页查看收藏。
9. 补充一个新菜品并上传图片。
10. 切换管理员账号，审核刚才提交的新菜品。
11. 回到食堂详情页，查看审核通过后的菜品。
