# 前端实现进度

## 第一次迭代：搭出基本页面

本次先把 Android 原生界面跑通。项目早期有一份 Web 原型，后面确定主要交付放在 `android` 目录。

- 实现了首页、美食广场、我的三个主页面。
- 实现了食堂详情和菜品详情两个详情页。
- 用 `MockRepository` 提供静态数据，先不接后端。
- 用 RecyclerView Adapter 展示食堂、菜品、评论、收藏等列表。

## 第二次迭代：调整信息架构和中文界面

这一轮主要修正交互顺序和界面文案。

- 将食堂名称改为紫荆、桃李、清芬、玉树、听涛、观畴、丁香。
- 首页公告改成浅蓝色细条滚动样式。
- 食堂详情页左侧改为楼层筛选，上方放搜索和窗口筛选。
- 图片暂时不接随机风景图，用空占位保持界面整洁。
- 标签按含义分色，例如辣味、清淡、甜品、面食使用不同颜色。

## 第三次迭代：补齐导航和列表交互

这一轮主要处理跳转和使用路径。

- 恢复底部“美食广场”入口，广场仍然负责展示所有食堂。
- 校园地图和热力图代码保留，但页面上先隐藏。
- 食堂详情和菜品详情增加顶部返回栏。
- 详情页也接入底部导航，便于快速回到首页、美食广场或我的。
- 我的收藏里的菜品可以点击进入菜品详情。

## 第四次迭代：表单和静态数据扩充

这一轮加了更接近完整应用的表单入口，但还没有接真实后端。

- 增加登录和注册界面，从“我的”页进入。
- “补充菜品信息”改为弹窗表单，包含文字说明、图片上传入口和标签输入。
- 菜品详情里的“写评价”改为弹窗表单，支持打分、写文字、填标签和选图片。
- 提交评价后会在当前菜品详情页即时显示一条新评价。
- 静态菜品数据扩充到 21 个，覆盖多个食堂和楼层。
- 美食广场的筛选标签也改为语义分色。

## 第五次迭代：调整启动流程和导航表现

这一轮主要按实际使用顺序调整入口。目前登录和注册还是静态演示，不做账号校验，也不保存用户数据。

- 将应用启动页改为登录/注册页，提交后再进入主界面。
- 登录和注册提交时给出静态演示提示，避免误以为已经接入真实账号系统。
- 底部导航栏改为只显示图标，图标放大，解决图标和文字挤在一起的问题。
- 写评价的打分控件从系统 `RatingBar` 改为普通五颗星选择。

## 第六次迭代：优化评分和账号操作

这一轮主要修正几个使用上不够顺的地方。

- 菜品打分不再使用系统 `RatingBar` 样式，改为五颗可点击的星星。
- 星星只有亮和不亮两种状态，亮星用橙黄色，避免出现半亮半不亮的视觉效果。
- 个人页去掉单独的登录/注册按钮，改为点击头像和用户名那一栏弹出账号操作。
- 账号操作弹窗提供“切换账号”和“退出登录”，都会进入静态登录页。
- 补充菜品和写评价的弹窗改成更大的自定义表单面板，用更圆的背景和更宽的间距。

## 当前架构

- `MainActivity` 负责底部三个主 tab。
- `HomeFragment` 负责首页公告和推荐内容。
- `FoodMapFragment` 负责食堂列表、搜索和 tag 筛选。
- `ProfileFragment` 负责个人页、收藏和账号操作入口。
- `CanteenDetailActivity` 和 `DishDetailActivity` 负责详情页。
- `AuthActivity` 负责静态登录/注册页面。
- `NavigationHelper` 统一处理底部导航跳转。
- `FormDialogs` 处理补充信息和写评价的弹窗表单。
- `MockRepository` 提供当前演示用的静态数据。
